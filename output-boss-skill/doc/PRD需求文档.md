# 软件台账管理平台 PRD 需求文档

## 1 项目概述

### 1.1 项目背景
随着企业软件资产日益增多，亟需建立统一的软件台账管理平台，实现软件资产的规范化管理。该平台主要包含软件台账和软件使用关系台账两大核心功能，支持多维度统计分析和可视化展示。

### 1.2 项目目标
- 建立统一的软件资产台账管理系统
- 实现软件使用关系的全面追踪
- 提供数据统计分析大屏
- 支持多角色权限管理

## 2 功能模块

### 2.1 软件台账

#### 2.1.1 软件数据模型
| 字段 | 说明 | 必填 |
|------|------|------|

| 软件功能类型（一级） | 办公/开发测试/运维管理/系统类等 | 是 |
| 软件功能类型（二级） | 如：办公类-文字处理 | 是 |
| 软件管理类型（介质类型） | 产品类/组件类 | 是 |
| 软件管理类型（仓库类型） | maven/npm/pypi等（仅组件类） | 否 |
| 软件负责人 | - | 是 |
| 开源协议 | 仅开源软件填写 | 否 |
| 使用限制 | 使用限制范围 | 否 |
| 软件功能介绍 | - | 否 |
| 软件使用手册 | - | 否 |

#### 2.1.2 软件版本（介质）数据模型
| 字段 | 说明 | 必填 |
|------|------|------|
| 软件ID | 关联的软件 | 是 |
| 软件版本 | - | 是 |
| 介质名称 | - | 是 |
| 介质下载地址 | - | 是 |
| 开源协议 | 仅开源软件填写 | 否 |
| 介质描述 | - | 否 |
| 介质类型 | 安装包/镜像/sdk等 | 是 |
| 介质大小 | - | 是 |
| 使用限制 | - | 否 |
| 使用说明 | 应用系统/终端使用说明 | 否 |

#### 2.1.3 数据来源
- 部分数据来源于引入流程
- 部分数据来源于其他系统同步，需展示同步状态

#### 2.1.4 页面设计
- **列表页**：展示软件基本信息和版本信息，支持多维度筛选、搜索、排序
- **详情页**：展示软件各个版本介质的详细情况，包括下载地址、介质类型、大小

#### 2.1.5 权限
无需登录，所有人均可查看（非敏感数据）

### 2.3 可观测需求

#### 2.3.1 软件数据统计大屏

**软件使用关系统计（所有人可查看）**
- 按终端/服务器/应用系统三个场景统计
- 团队负责人和超管可查看细致分类

**终端统计维度**
- 使用网段统计
- 终端所属团队统计
- 软件管理类型统计（开源/商业/自研）

**应用系统统计维度**
- 软件类型统计（产品类/组件类，组件类继续分maven/pypi等）
- 应用系统所属团队统计
- 软件管理类型统计
- 应用系统级别统计

**服务器统计维度**
- 软件类型统计（pkg/jar/pypi等）
- 软件管理类型统计
- 服务器所属应用系统所属团队统计
- 应用系统级别统计

**软件台账统计（所有人可查看）**
- 产品类/组件类统计
- 开源/商业/自研统计
- 功能类型统计（办公类/开发类/运维管理类/安全类等）

#### 2.3.2 同步情况（仅超管可查看）
- 同步状态统计：同步中/成功/失败
- 同步异常检测：报错展示、告警、数据不一致检测
- 同步失败详情：失败次数、失败原因
- 数据变动检测：新增/修改/删除统计

## 3 非功能需求

### 3.1 性能要求
- 支持大数据量查询（终端2000万、应用系统1000万+1000万）
- 分表分库策略：按软件类型分表（C1表按pkg/pypi/jar分表）

### 3.2 权限要求
- 基于角色的访问控制（RBAC）
- 数据行级权限控制

### 3.3 数据同步
- 多系统数据同步支持
- 同步状态实时监控
- 异常告警机制

---

## 4 验收标准

1. 软件台账列表页和详情页正常展示，支持筛选搜索
2. 软件使用关系台账三种类型页面正常展示，权限控制生效
3. 统计大屏数据准确，支持下钻
4. 同步检测功能正常，超管可查看同步状态
5. 系统响应时间满足性能要求
