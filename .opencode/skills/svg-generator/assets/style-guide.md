# SVG 图形生成规范

## 概述

本规范定义了 SVG 图形生成的标准，包括视觉风格、图形元素、箭头样式等，确保生成的 SVG 图形统一美观。

## 1. 基础规范

### 1.1 画布尺寸

| 类型 | 默认尺寸 | 最大尺寸 |
|------|---------|---------|
| 小型图 | 600×400 | 800×600 |
| 中型图 | 800×600 | 1000×800 |
| 大型图 | 1000×800 | 1200×1000 |

### 1.2 视图框

- 必须设置 `viewBox` 属性，与 width/height 保持一致
- 推荐使用 `viewBox="0 0 W H"` 格式

### 1.3 命名空间

```xml
<svg xmlns="http://www.w3.org/2000/svg" width="800" height="600" viewBox="0 0 800 600">
```

## 3. 箭头规范

### 3.1 箭头标记定义

```xml
<!-- 浅色模式箭头 -->
<marker id="arrowhead" markerWidth="6" markerHeight="4" refX="5" refY="2" orient="auto">
  <polygon points="0 0, 6 2, 0 4" fill="#999999"/>
</marker>

<!-- 浅色模式蓝色箭头 -->
<marker id="arrowhead-blue" markerWidth="6" markerHeight="4" refX="5" refY="2" orient="auto">
  <polygon points="0 0, 6 2, 0 4" fill="#3498db"/>
</marker>

<!-- 深色模式箭头（基于 examples/rd-lifecycle-flow.svg） -->
<marker id="arrowhead-dark" markerWidth="5" markerHeight="5" refX="5" refY="2.5" orient="auto">
  <polygon points="0 0, 5 2.5, 0 5" fill="#666"/>
</marker>
```

### 3.2 箭头样式

| 属性 | 标准值 | 说明 |
|------|-------|------|
| markerWidth | 6 | 箭头宽度 |
| markerHeight | 4 | 箭头高度 |
| refX | 5 | 箭头尖端偏移 |
| refY | 2 | 箭头垂直居中 |
| stroke-width | 2 | 连接线粗细 |

### 3.3 箭头颜色

- **灰色箭头**：使用 `#999999`（推荐用于连接线）
- **蓝色箭头**：使用 `#3498db`（可用于强调流程主线）
- **红色箭头**：使用 `#e74c3c`（用于错误分支）

### 3.4 线条样式规范

| 样式类型 | 使用场景 | stroke-dasharray | 颜色 |
|---------|---------|------------------|------|
| 实线 | 主线流程、正确分支、错误分支 | 无（默认） | `#999999` 或 `#e74c3c` |
| 虚线 | 仅用于异步/回调流程 | `5,5` | `#9b59b6` 或 `#999999` |
| 点划线 | 可选流程（不推荐） | `10,5` | `#999999` |

**重要原则**：
- 错误分支必须使用实线，用颜色（红色）区分，不可用虚线
- 虚线仅用于表示异步、回调、定时任务等非同步场景
- 避免过度使用虚线，保持图形简洁清晰

## 4. 图形元素规范

### 4.1 卡片/节点

```xml
<rect x="100" y="100" width="140" height="60" rx="8" ry="8"
      fill="#ffffff" stroke="#3498db" stroke-width="2"
      filter="url(#shadow)"/>
```

| 属性 | 标准值 | 说明 |
|------|-------|------|
| rx | 8 | 圆角半径 |
| stroke-width | 2 | 边框宽度 |
| filter | url(#shadow) | 阴影效果 |

### 4.2 阴影滤镜

```xml
<!-- 浅色模式阴影 -->
<filter id="shadow" x="-20%" y="-20%" width="140%" height="140%">
  <feDropShadow dx="0" dy="3" stdDeviation="6" flood-opacity="0.1"/>
</filter>

<!-- 深色模式阴影（基于实际示例） -->
<filter id="shadow-dark" x="-20%" y="-20%" width="140%" height="140%">
  <feDropShadow dx="0" dy="2" stdDeviation="4" flood-opacity="0.3"/>
</filter>
```

**注意**：深色模式示例中通常不使用阴影，或使用极淡的阴影效果。

### 4.3 标题文字

```xml
<text x="400" y="30" text-anchor="middle" 
      font-size="20" font-weight="bold" fill="#333333">
  图形标题
</text>
```

### 4.4 节点文字

```xml
<text x="170" y="130" text-anchor="middle" 
      font-size="14" font-weight="bold" fill="#333333">
  节点名称
</text>
```

## 5. 架构图规范

### 5.1 分层结构

架构图应采用分层设计，从上到下，例如，可根据用户需求自行调整：
1. 接入层（客户端、CDN、负载均衡、网关）
2. 服务层（业务服务、认证服务、消息队列）
3. 数据层（Redis、MySQL、MongoDB）
4. 基础层（Docker、K8s、监控）

### 5.2 图层标识

每层应有明确标识：
```xml
<text x="80" y="70" font-size="14" font-weight="bold" fill="#3498db">① 接入层</text>
```

### 5.3 分层容器

每层使用浅色边框容器，两侧留白：
```xml
<rect x="100" y="80" width="900" height="70" rx="8" 
      fill="#ffffff" stroke="#e0e0e0" stroke-width="1"/>
```

**布局原则**：
1. **上下对齐**：每层标签（如"① 接入层"）与容器左对齐，标签x坐标 = 容器x坐标
2. **居中分布**：容器内元素均匀居中分布，两侧保留100px边距
3. **模块间距**：模块间保持60-80px间距，确保视觉平衡
4. **避免占满**：容器宽度不超过画布宽度的90%，两侧留白
5. **底部区域协调**：底部补充区域（如数据流程、核心技术点）应与主架构区域宽度协调，保持左右两侧同等留白

**计算规则**：
```
容器宽度 = 画布宽度 × 0.9 (约900px)
容器起始x = (画布宽度 - 容器宽度) / 2 (约100px)
标签x坐标 = 容器起始x
模块总宽度 = ∑模块宽度
剩余空间 = 容器宽度 - 模块总宽度
间隔数 = 模块数 - 1
间隔宽度 = 剩余空间 / 间隔数
起始x = 容器x + (容器宽度 - (模块总宽度 + 间隔宽度×间隔数)) / 2

# 底部补充区域
底部区域宽度 = 容器宽度 / 2 - 20px (约430px)
左侧区域x = 容器起始x
右侧区域x = 容器起始x + 容器宽度 - 底部区域宽度
```

### 5.4 箭头使用规则

**分层架构图中箭头是可选的**，取决于图表目的：

1. **无箭头架构图**：当重点展示系统层次结构和组件分层时，可以不加箭头。适用于静态架构展示。

2. **带箭头架构图**：当需要表示数据流向、调用关系或处理顺序时，可添加箭头。箭头应使用统一灰色（`#999999`），大小保持一致。

3. **箭头样式**：
   ```xml
   <!-- 可选的层间箭头 -->
   <line x1="500" y1="150" x2="500" y2="175" 
         stroke="#999999" stroke-width="2" marker-end="url(#arrowhead)"/>
   ```

4. **决策依据**：
   - 如果用户描述中包含"流向"、"调用"、"请求"、"数据传输"等动态词汇，建议添加箭头。
   - 如果用户仅描述"架构"、"组成"、"层次"、"结构"等静态词汇，可不加箭头。

## 6. 流程图规范

### 6.1 画布尺寸计算规则

**核心原则：先计算内容高度，再确定画布尺寸，确保内容完整显示**

```
画布高度 = 标题高度(40) + 内容总高度 + 节点间距 × (节点数-1) + 边距(60-80) + 图例说明区(如有)
画布宽度 = 边距(40) + 最大节点宽度 × 列数 + 列间距 × (列数-1) + 边距(40)
```

**示例计算（纵向流程图）：**
- 5个节点，每个高度50px
- 节点间距：40px
- 标题+边距：80px
- 画布高度 ≥ 50×5 + 40×4 + 80 = 250 + 160 + 80 = 490px → 使用 600px

**画布尺寸选择建议：**
- 简单线性流程：800×500
- 带分支流程：900×650
- 复杂多分支：1000×800
- 架构图：1000×750

### 6.2 节点排列

- 横向流程：节点水平排列，间距 40-60px
- 纵向流程：节点垂直排列，间距 30-50px

### 6.3 连接线类型

#### 6.3.1 直线连接

```xml
<line x1="100" y1="130" x2="200" y2="130" 
      stroke="#999999" stroke-width="2" marker-end="url(#arrowhead)"/>
```

#### 6.3.2 折线连接（推荐用于分支流程）

```xml
<!-- 向下折线：从右向左的分支 -->
<path d="M 250 150 L 250 180 L 180 180 L 180 210" 
      fill="none" stroke="#999999" stroke-width="2" marker-end="url(#arrowhead)"/>

<!-- 向上折线：汇合流程 -->
<path d="M 180 300 L 180 330 L 250 330 L 250 360" 
      fill="none" stroke="#999999" stroke-width="2" marker-end="url(#arrowhead)"/>
```

#### 6.3.3 虚线连接（异步/可选流程）

```xml
<line x1="100" y1="130" x2="200" y2="130" 
      stroke="#999999" stroke-width="2" stroke-dasharray="5,5"
      marker-end="url(#arrowhead)"/>
```

#### 6.3.4 曲线连接（循环/回退）

```xml
<!-- 贝塞尔曲线：回退/重试流程 -->
<path d="M 250 200 C 250 250, 150 250, 150 300" 
      fill="none" stroke="#e74c3c" stroke-width="2" 
      stroke-dasharray="5,5" marker-end="url(#arrowhead)"/>
```

### 6.4 分支与汇聚

#### 6.4.1 菱形判断节点

```xml
<!-- 菱形节点：条件判断 -->
<polygon points="200,100 240,140 200,180 160,140" 
         fill="#fff" stroke="#e67e22" stroke-width="2" filter="url(#shadow)"/>
<text x="200" y="145" text-anchor="middle" font-size="12" fill="#333">判断?</text>
```

#### 6.4.2 分支连线标注

```xml
<!-- 分支连线旁标注条件 -->
<text x="200" y="160" text-anchor="middle" font-size="10" fill="#666">是</text>
<text x="200" y="160" text-anchor="middle" font-size="10" fill="#666">否</text>
```

#### 6.4.3 分支流程示例

```
        [开始]
          ↓
      ┌───────┐
      │ 判断? │
      └───┬───┘
        ↙   ↘
      [是]  [否]
        ↓     ↓
     [流程A] [流程B]
        ↘   ↙
      [汇聚点]
          ↓
        [结束]
```

#### 6.4.4 虚线框表示可选区域

```xml
<!-- 可选流程区域 -->
<rect x="50" y="200" width="200" height="120" rx="8" 
      fill="none" stroke="#999999" stroke-width="1" stroke-dasharray="5,5"/>
<text x="60" y="220" font-size="10" fill="#999">可选流程</text>
```

### 6.5 图形完整显示保障

**必须遵守以下规则：**

1. **边距计算**：画布各边保留 ≥30px 边距
2. **viewBox 匹配**：width/height 与 viewBox 完全一致
3. **内容边界检查**：绘制完成后验证所有节点在画布范围内
4. **文本不裁剪**：确保文字周围有足够padding
5. **箭头不截断**：连接线端点距节点边界 ≥5px

**调试技巧：**
- 临时添加画布边界线检查内容范围
- 使用 `<g id="debug">` 包裹临时调试元素，完成后删除

## 7. 稳健性设计规范

### 7.1 避免元素重叠和布局错乱

**1. 坐标计算规则：**
```
节点宽度 = 最大文字宽度 + 24px（左右边距）
节点高度 = 文字行数 × 行高 + 24px（上下边距）
水平间距 = max(节点宽度) × 1.2
垂直间距 = max(节点高度) × 1.1
```

**2. 节点排列算法：**
- 横向流程图：从左到右，水平对齐
- 纵向流程图：从上到下，垂直对齐
- 分支流程：父节点居中，子节点等距分布
- 汇聚流程：子节点对齐，汇聚点居中

**3. 避免重叠检查：**
- 节点间水平间距 ≥ 30px（流程图）、≥ 40px（架构图）
- 节点间垂直间距 ≥ 25px（流程图）、≥ 30px（架构图）
- 连接线不穿过其他节点
- 文字与图形边界间距 ≥ 8px
- 分支节点间距：分支展开角度合理，避免交叉

**4. 分支布局黄金法则：**
- 双分支：左右各偏转 30-45° 角
- 分支间距：在第一个转折点前保持 ≥ 50px 垂直距离
- 汇聚点：位于所有分支节点的公共下方区域
- 分支高度差：同一层级的分支节点底部应对齐

### 7.2 避免图形显示不完整

**1. 画布尺寸计算公式（紧凑原则）：**
```
画布宽度 = 边距(30-40)×2 + 内容总宽度 + 间距×(列数-1)
画布高度 = 标题区(60) + 内容总高度 + 间距×(行数-1) + 图例区(如有) + 边距(30-40)
```

**2. 画布尺寸选择指南（避免过大留白）：**
| 节点数 | 画布尺寸参考 |
|--------|-------------|
| 3-4个 | 600×300 ~ 700×400 |
| 5-7个 | 700×450 ~ 800×550 |
| 8-10个 | 800×550 ~ 900×650 |
| 10+个 | 按比例递增，每次增加约100px |

**3. 边距保障规则：**
- 所有元素距离画布边缘 ≥ 30px（简单图）、≥ 40px（复杂图）
- viewBox 必须与 width/height 完全一致
- 阴影、箭头等特效必须在 viewBox 范围内
- 标题距顶部边距 ≥ 25px

**4. 内容边界验证：**
- 最大 x 坐标 ≤ 画布宽度 - 30px
- 最大 y 坐标 ≤ 画布高度 - 30px
- 文字行不超出节点边界
- 折线转折点不与节点重叠

### 7.3 避免箭头混乱

**1. 箭头连接规则：**
- 连接线起点：从节点右侧中心或底部中心出发
- 连接线终点：指向节点左侧中心或顶部中心
- 折线转折点：至少距节点边界 15px
- 箭头大小统一：markerWidth=6, markerHeight=4
- 连接线端点距节点边界 ≥ 5px

**2. 箭头指向计算：**
```
向右连接：起点(x+width, y+height/2) → 终点(x-margin, y+height/2)
向下连接：起点(x+width/2, y+height) → 终点(x+width/2, y-margin)
向左连接：起点(x, y+height/2) → 终点(x+width+margin, y+height/2)
向上连接：起点(x+width/2, y) → 终点(x+width/2, y+height+margin)

其中 margin = 5px（箭头端点与节点边界的间距）
```

**3. 路径优化原则：**
- 优先使用直线连接
- 避免路径交叉
- 复杂路径使用 `<path>` 而非多段 `<line>`
- 箭头指向始终清晰，无歧义

**4. 折线路径设计规则：**
- 分支路径先垂直再水平（"L"形折线）
- 汇聚路径先水平再垂直
- 转折点坐标必须是整数
- 避免三条以上路径经过同一坐标点

**5. 连接线优先级：**
- 主线流程 > 次要流程 > 异步流程
- 实线用于同步流程和错误分支
- 虚线仅用于异步/回调场景
- 复杂路径使用 `<path>` 而非多段 `<line>`
- 箭头指向始终清晰，无歧义

### 7.4 错误处理规范

**1. 错误分支设计：**
- 使用红色实线，不可用虚线
- 错误节点使用红色边框
- 错误标签使用红色文字
- 错误流程清晰标注

**2. 虚线使用限制：**
- 仅用于异步流程（消息队列、回调、定时任务）
- 不用于错误分支、可选流程
- 异步流程使用紫色或灰色虚线

**3. 稳健性检查清单：**
- [ ] 无元素重叠
- [ ] 无路径交叉混乱
- [ ] 箭头指向准确
- [ ] 文字清晰不溢出
- [ ] 边距充足
- [ ] 颜色使用正确（错误分支=红色实线，异步=虚线）
- [ ] 图形显示完整，元素均在画布范围内

### 7.5 图例位置规范

**1. 图例放置原则：**
- 图例应置于图形**最底部**，避免与主流程重叠
- 图例位置：`transform="translate(x, y)"` 中的 y 值应为画布高度 - 图例高度 - 底部边距(40-60px)
- 图例宽度不宜超过画布宽度的 1/3，避免过于显眼
- **关键规则**：图例必须位于所有图形元素下方，确保无任何重叠

**2. 布局计算：**
```
图例区高度 = 图例内容高度 + 边距(20px)
画布高度 = 标题区(80) + 内容总高度 + 间距×(行数-1) + 图例区高度 + 底部边距(40)
图例Y坐标 = 画布高度 - 图例区高度 - 底部边距(40)
```

**3. 避免重叠规则：**
- 图例不得遮挡任何节点、连接线、文字
- 复杂流程图应预留图例区域，提前计算画布尺寸
- 图例背景使用浅色半透明，避免视觉干扰
- **异步流程检查**：异步操作节点（如日志记录）与图例保持安全距离，避免箭头重叠

**4. 画布尺寸与内容匹配原则：**
- 画布不应过大，避免大量空白
- 画布也不应过小，导致元素拥挤
- 建议画布利用率在 60%-80% 之间为宜
- 复杂图形可适当增大画布，但保持紧凑布局

**5. 示例位置：**
```xml
<!-- 图例置于底部右侧 -->
<g transform="translate(600, 650)">
  <rect width="200" height="120" rx="8" fill="#ffffff" stroke="#e0e0e0" stroke-width="1"/>
  <!-- 图例内容 -->
</g>
```

## 9. 验证清单

### 9.1 背景模式确认

生成前必须确认：
- [ ] 已询问用户需要浅色还是深色背景
- [ ] 背景色与内容颜色形成良好对比
- [ ] 深色模式下文字使用浅色系（#ffffff、#aaaaaa、#888888）

### 9.2 基础结构验证

- [ ] 箭头大小合适（浅色：markerWidth=6, markerHeight=4；深色：markerWidth=5, markerHeight=5）
- [ ] 箭头颜色匹配背景模式（浅色：#999999，深色：#666）
- [ ] 连接线粗细适中（stroke-width=2）
- [ ] 节点圆角一致（rx=5-8）
- [ ] 深色模式不使用阴影或使用极淡阴影（flood-opacity=0.1-0.3）
- [ ] 文字清晰可读，与背景对比度足够

### 9.3 布局规约验证

- [ ] 画布尺寸合适，无过多空白（利用率 60%-80%）
- [ ] 节点间距合理，无拥挤或过大间距
- [ ] 分支路径清晰，无路径交叉
- [ ] 折线转折点不与节点重叠
- [ ] 图例位于底部，与主流程无重叠

### 9.4 图形完整显示验证

- [ ] 图形能够完整显示，避免图形、元素显示不完整，包括背景、图例、说明等，不要超出画布范围
- [ ] 所有节点在 viewBox 范围内
- [ ] 箭头端点距节点边界 ≥5px
- [ ] 文字与图形边界间距 ≥8px

### 9.5 颜色与样式验证

- [ ] 颜色符合配色规范
- [ ] 错误分支使用红色实线（非虚线）
- [ ] 异步流程使用虚线（紫色或灰色）
- [ ] 边框宽度合适（浅色模式：2px，深色模式：2-3px）
