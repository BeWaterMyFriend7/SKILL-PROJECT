# SVG 图形生成器

## 核心工作流

```
用户描述 → 解析需求 → 设计图形 → 生成 SVG → 验证 → 保存文件
```

## 执行步骤

### 第 1 步：解析需求

确定图形类型、布局方向、内容、背景模式（浅色/深色）。
**关键**：未明确背景模式时默认浅色。

### 第 2 步：生成图形

参考示例文件：`examples/*.svg`

### 第 3 步：验证清单

- [ ] 背景模式正确
- [ ] 节点无重叠，间距合理
- [ ] 连接线端点精确指向节点边缘（距边界≥5px）
- [ ] 画布利用率 60%-80%
- [ ] 箭头样式统一，不交叉
- [ ] 错误分支用红色实线，异步流程用虚线
- [ ] 所有元素在 viewBox 范围内

# 图形类型规范

## 架构图

分层结构：接入层 → 服务层 → 数据层 → 基础层
- 每层内元素等宽，左右边距对称，垂直居中于容器
- 层标签在容器上方，颜色与层功能匹配
- 支持：应用架构图、部署架构图、系统架构图、业务架构图

## 对比图

左右对称布局，左侧蓝色系、右侧绿色/橙色系，中间"VS"标识。

## 时间线图

横向时间线从左到右，节点用圆形或卡片，每个时间点包含标签和事件描述。

## 状态流转图（重要）

节点类型：初始→实心圆 / 结束→双圆(绿) / 状态→圆角矩形(rx=6) / 判断→菱形

**连接线规范：**
- 优先水平/垂直直线，避免斜线
- 分支用折线（先直线走到分支点再转向）
- 条件标注在箭头旁，不跨其他元素
- 所有状态节点网格对齐，连接线不穿过节点

## 时序图（重要）

- 生命线等距分布（间距 200-250px）
- 消息从上到下按时间顺序排列，水平直线
- 激活条顶部对齐消息起点
- 箭头向右=调用，向左=返回

## ER图

- 实体：圆角矩形带顶部标题栏
- 属性：连接实体的小圆角矩形
- 关系：菱形连接符
- 主键加粗/PK标识，外键斜体/FK标识

## 文章总结图

卡片流程式布局（左到右），每卡包含步骤编号、标题、描述，箭头连接步骤。底部注意事项框用虚线边框。

# 布局规范

### 节点尺寸
- 菱形：60×40px | 矩形：140-160×50-60px | 步骤卡：160×100px
- 圆形：16-20px(状态点) / 12px(时间点)

### 间距
- 垂直：50-60px | 水平：40-60px | 分支点前：≥50px

### 画布计算
```
画布高度 = 标题40 + 内容总高 + 间距×(节点数-1) + 图例120 + 边距40
画布宽度 = 边距40 + 最大宽×列数 + 列间距×(列数-1) + 边距40
```
