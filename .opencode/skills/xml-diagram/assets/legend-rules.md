# 图例规则

图例用于解释颜色和线型语义。满足触发条件时必须生成。

## 必须生成图例的条件

```text
使用 4 种以上语义色。
存在实线、虚线、主流程、错误线等线型区别。
存在阶段颜色。
存在角色颜色。
存在系统边界颜色。
```

## 位置优先级

按以下顺序选择不会遮挡主内容的位置：

```text
1. 右侧独立图例区。
2. 底部横向图例区。
3. 右下角悬浮图例。
```

禁止：

```text
图例覆盖主内容。
图例放在主流程中间。
图例与节点距离小于 24px。
满足触发条件却省略图例。
```

## 颜色图例

默认语义：

```text
蓝色 = 用户 / 前端 / 需求 / 主体
绿色 = 服务 / 能力 / 成功
橙色 = 外部系统 / 设计 / 警告
红色 = 错误 / 风险 / 失败
紫色 = 测试 / 异步 / 对比对象
青色 = 部署 / 发布 / 通信 / 存储访问
灰色 = 基础设施 / 中性边界
```

## 线型图例

出现多种线型时使用：

```text
实线箭头 = 同步调用 / 普通流程
蓝色粗实线 = 主流程
紫色虚线 = 异步消息 / 事件流 / 回调
红色实线 = 异常 / 失败路径
青色实线 = 存储访问
灰色实线 = 运维观测
```

## 阶段图例

项目时间线和生命周期图使用：

```text
需求
立项
设计
开发
测试
部署
运维
迭代
```
