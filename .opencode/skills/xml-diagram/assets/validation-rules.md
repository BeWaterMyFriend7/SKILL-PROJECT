# 校验规则

生成 drawio XML 后必须校验。校验不通过时先修正，再交付。

```text
中间结构 -> 布局 -> 配色 -> 连线 -> 校验 -> XML
                                  ↑ 失败则修正
```

## 颜色校验

```text
所有颜色来自标记表或用户明确品牌色。
同一张图语义色不超过 8 种。
正文文字与背景对比清晰。
深色模式禁止白色卡片填充。
深色模式禁止阴影。
浅色语义色块不能使用低对比白字。
```

## 标题安全区校验

```text
每个区域必须有标题安全区。
标题区高度至少 44px。
区域内节点不得进入标题安全区。
节点 y 坐标必须大于 区域 y + 标题高度 + 上内边距。
```

## 重叠校验

```text
区域标题与卡片不重叠。
卡片与卡片不重叠。
标签与标签不重叠。
图例与主内容不重叠。
箭头标签不遮挡节点。
任意两个元素最小间距至少 24px。
分组卡片间距至少 28px。
区域间距至少 36px。
```

## 边距校验

```text
所有元素在画布范围内。
所有内容距画布边缘至少 30px。
标题距顶部至少 20px。
图例和脚注必须计入整体边界。
```

## 画布利用率校验

```text
内容区域占画布 60%-80% 为目标。
低于 50% 时缩小画布或重新居中。
高于 85% 时扩大画布、换行或拆分区域。
不得留下大面积无意义空白。
```

低利用率处理顺序：

```text
先判断是否内容不足。
内容不足时补充节点、标签和说明，不要只缩小画布。
内容足够但分散时重排网格。
重排后仍低于 50% 才缩小画布。
```

## 架构语义密度校验

```text
架构图不能只靠少量大矩形撑满画布。
区域标题必须是纯文本，不得有单独标题矩形框、填充底色或按钮式边框。
第 3 层内容必须是小矩形标签，不能是卡片内部纯文字、换行列表或项目符号。
业务架构图必须覆盖场景、能力、对象/资源三类信息。
应用、技术、部署架构图必须覆盖入口、核心处理、数据/资源、外部依赖或治理观测等关键维度。
如果语义密度不足，先补充内容或询问用户，再调整画布。
架构图生成前必须存在内容充实记录：确定内容、合理假设、需要确认的内容。
```

## 时序结构校验

```text
时序图必须有横向对齐的参与者。
每个参与者必须有垂直生命线。
多于 3 条消息时必须有激活条。
激活条必须是窄矩形，贴近对应生命线，表示处理持续时间。
消息必须按 y 坐标自上而下排列，返回消息使用虚线。
禁止把时序图画成流程图卡片链。
```

## 编码校验

```text
所有 drawio、Markdown 和脚本文件必须是严格 UTF-8。
禁止用 PowerShell Get-Content / Set-Content 改写中文 XML。
禁止把 UTF-8 字节按 GBK 解码后再写回。
发现常见中文 mojibake 片段或 Unicode 替换字符时，视为编码损坏。
编码损坏的文件不得继续局部替换，应从可信模板或原始语义重新生成。
```

## 连线校验

```text
默认使用正交连线。
禁止曲线。
禁止斜线。
优先使用节点起点和终点引用。
禁止穿过分组卡片。
跨区域连线从容器边缘连接。
错误路径为红色实线。
异步路径为紫色虚线。
虚线不得用于错误路径。
主流程不得与普通连线视觉混淆。
```

## 简约步骤流程校验

```text
主步骤数量不超过 8 个。
箭头数量不超过主步骤数量减一。
禁止复杂分支、回环和交叉线。
两行布局最多允许一条换行连接线。
条件判断不单独展开成复杂分支。
```

## 图例校验

```text
满足图例触发条件时必须有图例。
图例不得遮挡主内容。
图例与节点距离至少 24px。
图例必须解释颜色或线型语义。
图例内容不得与实际图形语义冲突。
```

## 圆角一致性

```text
同级卡片圆角一致。
区域容器圆角大于卡片圆角。
小标签圆角小于卡片圆角。
不要用多个圆角形状叠加造成边界错位。
```

## XML 校验

```text
XML 必须能被 drawio 打开。
所有 mxCell 必须有唯一标识。
所有 mxGeometry 必须包含 as="geometry"。
背景、连线、标签、容器和普通节点的 mxGeometry 都必须检查。
所有连线引用的起点和终点必须存在。
XML 注释不得包含 --。
pageWidth 和 pageHeight 必须覆盖全部内容。
```

## 校验顺序

```text
1. XML 结构校验
2. UTF-8 编码和乱码校验
3. mxGeometry as="geometry" 校验
4. 颜色校验
5. 深色模式校验
6. 标题安全区校验
7. 重叠校验
8. 边距校验
9. 画布利用率校验
10. 架构语义密度校验
11. 时序结构校验
12. 连线校验
13. 圆角一致性校验
14. 图例校验
```

## 失败处理

```text
发现错误时先修正布局或样式，再重新校验。
不要通过删除必要内容来逃避校验。
不要为了通过校验牺牲图形语义。
若内容过多，优先扩展画布、分组或换行。
```
