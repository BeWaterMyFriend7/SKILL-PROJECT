# 连线规则

所有连线必须由连线计划统一生成，禁止临时手画混乱路径。

## 连线类型

| 类型 | 颜色 | 样式 | 场景 |
| --- | --- | --- | --- |
| 普通 | 默认连线色 | 实线 | 普通流程、数据连接 |
| 主流程 | 蓝色 | 粗实线 | 主链路、核心处理路径 |
| 异步 | 紫色 | 虚线 | 消息队列、事件流、回调 |
| 错误 | 红色 | 实线 | 错误、异常、失败路径 |
| 成功 | 绿色 | 实线 | 成功、通过、结束 |
| 返回 | 灰色或紫色 | 时序图使用虚线，普通流程可用实线 | 返回消息、响应 |
| 存储 | 青色或灰色 | 实线 | 数据库、缓存、文件、搜索访问 |
| 观测 | 灰色 | 实线 | 日志、监控、追踪、告警 |
| 生命线 | 默认连线色 | 细虚线 | 时序图生命线 |

## 基础样式

```text
使用正交连线。
禁止曲线。
禁止斜线。
优先使用节点起点和终点引用。
连线端点距节点边界至少 5px。
同一图中箭头大小统一。
连线标签背景与画布背景一致。
```

## 路径避让

```text
禁止连线穿过分组卡片。
跨区域连线必须从区域边缘连接。
箭头标签不得遮挡节点或图例。
三条以上连线不得经过同一坐标点。
出现交叉时优先调整布局，其次调整转折点。
```

## 错误路径

```text
错误路径必须使用红色。
错误路径必须使用实线。
错误路径禁止使用 dashed=1。
此规则不可被用户风格要求覆盖。
```

## 异步路径

```text
异步、回调、消息队列、事件流使用虚线。
虚线默认使用紫色。
虚线不得用于错误路径。
虚线不得用于主流程。
```

## 主流程

```text
主流程必须比普通线更突出。
推荐蓝色粗实线。
主流程只表达核心业务链路，不要把所有连接都做成主流程。
```

## 简约步骤流程图连线

```text
只保留主步骤之间的必要箭头。
箭头数量最多为主步骤数量减一。
优先使用短直线或短正交线。
两行布局最多允许一条换行连接线。
禁止复杂分支、回环和交叉线。
条件说明优先放入卡片文字，不画菱形分支。
```

## 时序图连线

```text
参与者之间的请求线必须水平对齐。
同步请求使用实线箭头，返回响应必须使用虚线箭头。
生命线使用浅灰或深色弱线，不使用主流程箭头样式。
多于 3 条消息时必须绘制激活条；激活条贴近生命线中心，高度覆盖一次调用的持续区间。
消息标签放在线条上方 8-14px，不进入参与者标题框。
同一条消息线的 source 和 target 优先引用参与者或生命线节点。
禁止用流程图的折线箭头、菱形判断和步骤编号卡片表达时序。
```

## 图例触发

同一图中出现任意两种以上线型语义时，必须生成线型图例。
