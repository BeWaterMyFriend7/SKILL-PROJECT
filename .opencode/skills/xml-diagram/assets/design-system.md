# 设计系统

## 目标

生成的 drawio XML 必须可编辑、可复用、风格统一。图形逻辑可以随用户输入变化，但视觉结构、间距、标题层级、卡片层级、颜色、图例和连线语义必须保持一致。

## 基本原则

```text
逻辑由模型规划。
样式由规约约束。
坐标由模板和布局规则控制。
质量由校验器兜底。
```

## 语义色

| 角色 | 颜色 | 用途 |
| --- | --- | --- |
| 主色 | `#3498db` | 用户、前端、需求、主节点 |
| 成功 | `#2ecc71` | 服务、能力、通过、成功 |
| 警告 | `#e67e22` | 外部系统、设计、警告 |
| 错误 | `#e74c3c` | 错误、风险、失败 |
| 紫色 | `#9b59b6` | 异步、队列、测试、对比对象 |
| 青色 | `#1abc9c` | 部署、发布、通信、存储访问 |
| 灰色 | `#95a5a6` | 基础设施、中性边界 |

同一张图的语义色不超过 8 种。

## 禁止项

- 错误分支使用虚线。
- 虚线用于同步调用或主流程。
- 连线穿过卡片主体。
- 节点覆盖区域标题。
- 大面积背景使用高饱和语义色。
- 图例覆盖主内容。
- 画布利用率低于 50% 仍不收缩画布。
