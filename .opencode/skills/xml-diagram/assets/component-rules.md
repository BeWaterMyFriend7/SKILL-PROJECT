# 组件规则

每张图都必须先明确组件层级。不要把不同层级的元素做成同一种视觉样式。

## 第 0 层：页面

```text
页面 = 主标题 + 可选副标题 + 主内容 + 图例或脚注
```

## 第 1 层：区域容器

用于大领域、分层、区域、阶段或网络边界。

示例：

```text
业务场景层
业务能力层
业务对象层
前端应用层
后端服务层
外部系统层
关键里程碑
数据与存储层
运维与观测层
```

```yaml
圆角: 12
边框宽度: 2
填充: 容器背景
左右内边距: 28
上下内边距: 24
标题区高度: 44
```

禁止：

```text
节点进入区域标题区。
区域没有内部留白。
多个区域间距小于 32px。
把区域标题、分层标题或一级内容标题画成独立矩形框。
给一级标题添加填充底色、边框或按钮式背景。
```

区域标题必须是纯文本，放在容器标题安全区内；区域容器本身可以有背景和边框，但标题文字本身不要再套一层矩形。

## 第 2 层：分组卡片

用于领域、模块、阶段、服务、系统或对象组。

示例：

```text
用户管理域
商品管理域
接口网关
会话管理
设计阶段
开发阶段
```

```yaml
宽度: 180-260
高度: 自动
最小高度: 86
圆角: 10
边框宽度: 2
标题高度: 34
正文内边距: 12
```

当卡片内部包含多个小标签时，应保留清晰的标题区域。

## 第 3 层：小标签

用于能力点、技术点、属性和短事实。

示例：

```text
注册登录
信息管理
服务框架
缓存组件
监控采集
```

```yaml
宽度: 自动
最小宽度: 54
高度: 22
横向间距: 8
纵向间距: 8
圆角: 6
填充: 浅色模式使用白色或浅语义底色，深色模式使用深色标签底色
边框: 必须存在，颜色弱于分组卡片边框
```

架构图中的第 3 层内容必须绘制为独立的小矩形标签。不要把“注册登录 / 信息管理 / Redis / 监控采集”这类内容直接写成卡片里的纯文字、换行列表或项目符号。每个小标签都必须是可编辑的 `mxCell`，包含 `mxGeometry as="geometry"`。

## 第 4 层：说明

用于备注、脚注和弱标签。说明文字不得抢占卡片标题的视觉层级。

## 图例组件

图例必须作为结构化组件绘制，不要散落成普通文本。图例可包含颜色块、线型样例或阶段标记。
