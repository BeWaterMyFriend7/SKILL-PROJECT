## 14. 执行指引

> **批准后**，Agent 直接按本章执行，无需加载其他 Skill。

### 14.1 执行前置校验

修改任何代码前，必须逐项确认：

| 检查项 | 期望值 | 实际值 | 状态 |
|---|---|---|---|
| 本报告审批状态 | `APPROVED` | | |
| HEAD | 与第 1 章一致 | | |
| Git 操作对象 | 与第 1 章一致 | | |
| 未解决冲突文件集合 | 与计划一致 | | |
| 各冲突文件 stage-1/2/3 Blob | 与 JSON 一致 | | |
| 工作区无未知改动 | 无 | | |

> 任一不一致 → **立即停止**，报告 `PLAN_DRIFT`，要求重新运行本 Skill 分析。

### 14.2 执行顺序与约束

推荐修改顺序：**源定义/接口 → 业务实现 → 配置/迁移 → 生成文件/锁文件 → 测试 → 文档**

硬约束：
- **默认不执行** `git add`、`git commit`、`git push`（除非审批人明确要求）
- **禁止** `git reset --hard`、`git clean -fd`、强制 checkout 或删除分支
- **禁止** 用 `checkout --ours/--theirs` 替代报告中的语义决策
- **禁止** 修改白名单（第 6 章）之外的任何文件
- **禁止** 删除有效断言、降低校验标准或静默改变公共行为以通过测试
- 每修改一个文件后运行 `git diff --check` 确认无残留标记

### 14.3 范围守卫

每完成一个执行步骤后运行：
```bash
python scripts/guard_change_scope.py --plan <merge-plan.json> --repo .
```
出现白名单外文件 → **立即停止**，生成修正请求。

### 14.4 分层验证（按成本从低到高）

| 层级 | 检查内容 | 必须执行 |
|---|---|---|
| 1. 结构检查 | `git diff --check`、残留冲突标记 `<<<<<<<`/`=======`/`>>>>>>>`、未合并条目 `git diff --name-only --diff-filter=U` | ✅ 是 |
| 2. 静态检查 | 编译、格式化、lint、类型检查、代码生成一致性 | ✅ 是 |
| 3. 针对性测试 | stage-2 行为、stage-3 行为、融合路径的专项测试 | ✅ 是 |
| 4. 集成检查 | API、配置、跨模块调用、依赖树 | 按计划要求 |
| 5. 全量验证 | 完整测试套与构建 | 成本允许时执行 |
| 6. 人工检查 | 发布、灰度、监控、外部系统 | 无法自动化时记录 |

优先运行：
```bash
python scripts/verify_merge_state.py --repo . [--command "..."]
```
然后逐项执行第 9 章的验证项。**不得将未执行标记为通过。**

### 14.5 偏差处理

执行中出现以下情况属于偏差，**不得现场自由发挥**：
- 需要修改白名单外文件
- 方案依赖的符号/接口不存在
- 两个批准动作互相矛盾
- 测试证明方案语义错误（而非简单实现错误）
- 出现新的数据库、安全、兼容或发布风险

此时使用 `assets/plan-amendment.md` 模板生成修正请求，包含：
- 触发原因
- 原决策 vs 拟变更
- 新增影响文件
- 新验证项
- 风险变化

**修正批准前停止一切修改。**

### 14.6 最终差异复核

全部动作完成后检查：
- `git diff --stat` 只在白名单范围内
- 每个行为保留项（第 5 章）能映射到实际代码和验证结果
- 无残留临时代码、调试日志、跳过标记或 TODO
- 无删除任一侧有效测试、配置或文档更新
- 生成文件和锁文件可由正式工具重现

### 14.7 输出执行结论

使用以下状态之一：
- **`VERIFIED`**：全部批准动作完成，所有必需验证通过
- **`IMPLEMENTED`**：代码已实施，存在明确列出的未执行验证
- **`BLOCKED`**：现场漂移、范围扩张、必需验证失败或出现新业务决策

执行报告（写入 `.git/codex-merge-review/<plan-id>/merge-execution-report.md`）必须列出：
- 实际修改文件、每个动作状态、每个验证项命令与结果
- 双侧行为保留结果、偏差与修正记录
- 未验证项、残余风险和回滚建议
- 是否执行了 `git add/commit/push` 及原因
