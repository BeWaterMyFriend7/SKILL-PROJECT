# 报告 → 执行 交接契约

## 单一产物

Markdown 报告是唯一的产出文件，包含：
- 第 1-12 章：说明为什么这样合并，供人工审批
- 第 14 章：执行指引，供 Agent 直接执行
- 附录 A：JSON 执行契约（fenced code block），供脚本做确定性校验

报告与附录 A 共享同一 `plan_id`，由同一文件构造保证一致性。

## 防止旧报告误用

附录 A JSON 必须记录：

- 仓库根目录。
- HEAD。
- Git 操作类型和操作对象提交。
- 未解决冲突文件集合。
- 每个冲突文件的 stage-1/2/3 Blob ID。

Agent 执行前（按报告第 14.1 节）逐项比对。任何关键漂移都应重新分析，而不是尝试自动适配。

## 审批边界

Planner 只能输出 `WAITING_FOR_APPROVAL`。批准后应明确：

- 全部批准，或批准的决策/动作 ID。
- 审批人和时间。
- 修改文件边界。
- 是否允许暂存、提交或运行高成本验证。

审批不得隐含在提交信息或模糊对话中。
