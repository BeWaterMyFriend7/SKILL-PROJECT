# 合并验证矩阵

| 变更类型 | 最低验证 | 重点问题 |
|---|---|---|
| 普通业务代码 | 编译/类型检查、冲突模块单测、双方行为用例 | 分支条件、异常、空值、边界 |
| 公共 API | 契约测试、消费者兼容、序列化测试 | 字段必选性、状态码、默认值、版本 |
| 配置 | 配置解析、默认值、各环境覆盖 | key 丢失、类型变化、废弃项、密钥 |
| 数据库迁移 | 干跑/测试库升级、顺序与幂等检查 | 已发布迁移改写、锁表、回滚、滚动升级 |
| 消息/事件 | schema 校验、生产者消费者联调 | 字段兼容、重复消费、顺序、重试 |
| 并发代码 | 并发测试、竞态检测、超时与取消 | 锁顺序、资源泄漏、重复执行 |
| 依赖/锁文件 | 重新生成锁文件、依赖树检查、构建 | 版本漂移、重复依赖、平台差异 |
| 生成代码 | 运行生成器、工作区无生成差异 | 源定义与生成结果不一致 |
| CI/脚本 | 语法检查、最小执行、关键参数验证 | shell 差异、权限、路径、环境变量 |
| 文档/示例 | 示例执行或链接检查 | 文档与最终接口不一致 |
| 删除/重命名 | 全局引用搜索、构建、兼容入口检查 | 隐式加载、反射、配置引用、部署脚本 |

## 双侧行为验证

每个推荐方案至少设计三类用例：

1. stage-2 新增或修复行为仍成立。
2. stage-3 新增或修复行为仍成立。
3. 两者同时出现时的融合行为正确。

只运行现有测试但没有覆盖双方意图时，不得判定语义合并完成。
