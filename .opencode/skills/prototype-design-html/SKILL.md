# 原型设计

## 执行步骤

### 第 1 步：读取需求

- 读取 `output/doc/requirements-analysis-review.md`
- 或读取用户提供的需求文档
- 无文档时：先进行需求分析

### 第 2 步：原型设计

在 `output/design-output/` 下输出：
- 可交互的 HTML 文件
- 配套资源文件
- 项目 README 手册

### 第 3 步：验证测试

对 HTML 原型进行交互验证，确保无问题

## 设计规范

### 基础要求
- 简约大气，避免过于花哨
- 参考 `reference/design-rule.md`
- 仅设计 Web 端，16:9 比例

### 配色体系（参考共享设计系统）
| 用途 | 颜色 | 说明 |
|------|------|------|
| 页面背景 | #f8f9fa | 浅灰底 |
| 卡片/容器 | #ffffff | 白色填充 |
| 默认边框 | #e0e0e0 | 浅灰边框 |
| 主色调 | #3498db | 蓝色，按钮/链接/强调 |
| 成功色 | #2ecc71 | 绿色，成功状态 |
| 警告色 | #e67e22 | 橙色，警告提示 |
| 错误色 | #e74c3c | 红色，错误/删除 |
| 主文字 | #333333 | 标题/正文 |
| 次要文字 | #666666 | 辅助说明 |

### 字体层级
| 层级 | 大小 | 粗细 | 用途 |
|------|------|------|------|
| H1 | 24px | bold | 页面标题 |
| H2 | 18px | bold | 区块标题 |
| H3 | 16px | bold | 卡片标题 |
| Body | 14px | normal | 正文内容 |
| Caption | 12px | normal | 辅助说明 |

### 间距系统
- 页面边距：30-40px
- 卡片内边距：16-20px
- 组件间距：20-30px
- 卡片圆角：6-8px

### 数据持久化
- 优先使用 `localStorage` 存储用户操作状态
- 复杂数据使用 `JSON.stringify`/`JSON.parse`
- 不使用外部数据库或后端服务
- 必要时提供 mock 数据初始化

## 输出

- 可运行的 HTML 文件
- 交互流畅无明显 bug
