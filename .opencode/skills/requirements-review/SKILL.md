# 需求评审

## 执行步骤

### 第 1 步：读取需求文档

- 用户提供：从用户提供的位置读取
- 未提供：读取 `output/doc/requirements-analysis.md`
- 无文档：要求用户提供

### 第 2 步：进行评审

指出以下问题：
- 需求不明确的地方
- 需求不完整的地方
- 交互存在问题的点
- 有歧义的内容

### 第 3 步：确认细节

对评审中发现的问题，引导用户确认：
- 不明确的内容需要明确
- 多种解决方案给出建议

### 第 4 步：输出评审报告

保存至 `output/doc/requirements-analysis-review.md`

包含：评审发现的问题、修改建议、本次调整内容

## 输出规范

- 使用 Markdown 格式
- 内容简洁明了
- 列出具体问题和修改建议
