# 需求分析

## 执行步骤

### 第 1 步：需求理解

接收用户需求描述，理解业务目标、用户群体、核心功能

### 第 2 步：要点拆解

拆解关键需求点：
- 功能需求
- 非功能需求（性能、安全、兼容性等）
- 用户体验要求
- 边界条件

### 第 3 步：确认细节

对不明确的内容引导用户回答：
- 需求细节不清楚的地方
- 多种解决方案可选时，给出建议请用户确认

### 第 4 步：输出文档

将需求分析文档保存至 `output/doc/requirements-analysis.md`

参考示例：`example/requiremet-ex.md`

## 输出规范

- 使用 Markdown 格式
- 内容简洁明了
- 包含：需求概述、功能列表、用户流程、非功能需求
