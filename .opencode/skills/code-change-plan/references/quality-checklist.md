# 方案质量检查清单

> 配合 SKILL.md 使用。方案输出前必须逐项核对。任何一项不满足，都应继续完善方案。

## 检查项

1. **需求表达** — 是否准确表达用户需求？不得偏离或自行扩大范围。
2. **流程区分** — 当前流程和目标流程是否清晰区分？不得混为一谈。
3. **项目路径** — 是否使用准确的项目路径？不得使用占位符或猜测路径。
4. **全项目范围** — 是否分析了所有相关项目范围？不得只检查用户点名的文件。
5. **旧资产处理** — 每个被替换的旧资产是否有明确处理方式（删除/替换/迁移/兼容）？
6. **双逻辑检查** — 是否可能意外保留新旧双逻辑？新方案不得与旧实现同时生效。
7. **文件动作** — 每个文件的修改动作是否明确（修改/新增/删除/重命名）？
8. **实施顺序** — 实施顺序是否合理？是否按依赖关系排列？
9. **验证命令** — 验证命令是否适用于当前项目？是否写明了期望结果？
10. **追踪完整性** — REQ → AC → CU → 文件 → V 和 LR → CU → V 是否完整？
11. **停止条件** — 是否包含停止条件？执行中遇到什么情况应中止？
12. **审批状态** — 审批状态是否为 `WAITING_FOR_APPROVAL`？
13. **不修改代码** — 是否确认没有修改任何业务代码？
14. **章节完整** — 所有必需章节是否齐全？不适用的章节是否标注了原因？
