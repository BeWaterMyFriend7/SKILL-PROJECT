# 需求要点
1、设计软件台账管理平台，主要包含软件台账和软件使用关系台账两个大的功能点。
2、软件台账包含产品类、组件类使用关系台账，支持各类维度的筛选
3、软件台账分为列表页和详情页，详情页为软件各个版本介质的情况展示
4、软件使用关系台账分为终端类、应用系统类、服务器三大类，每类又有小类别
5、可观测需求：软件数据的统计、同步情况和大屏展示
6、权限需求：根据不同的角色可查看不同的数据

# 需求详细描述
## 1、软件台账
### 1.1 软件数据模型
- 软件的数据模型
  - 软件id
  - 软件名称
  - 软件管理类型：商业软件、开源软件、自研软件
  - 软件功能类型（一级）：办公、开发测试、运维管理、系统类等
  - 软件功能类型（二级）：办公类-文字处理、办公类-多媒体处理等
  - 软件管理类型（介质类型）：产品类、组件类
  - 软件管理类型（仓库类型，仅组件类）：maven、npm、pypi等
  - 软件负责人
  - 开源协议（仅开源软件）
  - 使用限制：使用限制范围
  - 软件功能介绍
  - 软件使用手册
- 软件版本的数据模型
  - 管理的软件id
  - 软件版本
  - 介质名称
  - 介质下载地址
  - 开源协议（仅开源软件）
  - 介质描述
  - 介质类型：安装包、镜像、sdk等
  - 介质大小
  - 使用限制：使用限制范围
  - 使用说明：应用系统、终

### 1·2 数据来源
- 部分数据来源于引入流程
- 部分数据来源于其他系统同步，需要在同步的检测中有所展示

### 1.3 页面
- 列表页：展示软件基本信息和版本信息，支持多维度筛选、搜索、排序
- 详情页：展示软件各个版本介质的详细情况，包括下载地址、介质类型、大小，支持筛选、排序和搜索

### 1.4 权限
 这部分数据为非敏感数据，无需登录均可查看

## 2、软件使用关系台账
### 2.1 数据模型
- 终端使用关系数据模型 A1 数据量 2kw
  - 终端MAC（与终端设备数据管理，获取终端相关数据）
  - 终端ip
  - 终端使用软件名称
  - 终端使用软件版本
  - 软件是否合规
  - 软件是否为黑名单软件
  - 终端使用软件安装路径
  - 终端使用软件安装时间
  - 数据状态：正常、残留数据（有很多历史数据由于采集的原因，无法采集准确，只能依赖用户反馈）、未知
- 终端设备数据模型 A2  一条A2记录对应多条A1记录 数据量 10w
  - 终端MAC
  - 终端ip
  - 终端网段：办公、开发测试、生产
  - 终端名称
  - 终端负责人
  - 终端负责人团队
  - 终端状态：正常、已回收、未知

- 应用系统使用关系数据模型-组件类 B1 数据量 1kw
  - 应用系统名称
  - 应用系统使用软件版本
  - 应用系统使用软件名称
  - 软件是否合规
  - 软件是否为黑名单软件
  - 应用系统使用软件管理类型（介质类型）：产品类、组件类
  - 应用系统使用软件管理类型（仓库类型，仅组件类）：maven、npm、pypi等
  - 应用系统使用软件开源协议
-  应用系统使用关系数据模型-产品类 B2 数据量 1kw
  - 应用系统名称
  - 应用系统使用软件版本
  - 应用系统使用软件名称
  - 软件是否合规
  - 软件是否为黑名单软件
  - 所在服务器IP
  - 应用系统使用软件管理类型（介质类型）：产品类、组件类
  - 应用系统使用软件开源协议
- 应用系统数据模型 B3 数据量 1w
  - 应用系统名称 （与B1、B2 关联）
  - 应用系统负责人
  - 应用系统团队负责人
  - 应用系统类型
  - 应用系统级别
  
- 服务器使用关系数据模型 C1 
  - 服务器IP 
  - 服务器名称
  - 服务器类型
  - 安装软件名称
  - 安装软件版本
  - 软件是否合规
  - 软件是否为黑名单软件
  - 安装软件位置
  - 安装软件类型（pypi（500w）、pkg（2000w）、jar（2000w），每个类型一张表，分开查询存储，）
  - 其他字段（根据软件类型的不同有不同的字段）
- 服务器数据模型 C2
  - 服务器IP（与C1关联）
  - 服务器所属应用系统（与B3关联）
  
### 2.2 数据来源
    数据来源于其他系统同步，需要在同步的检测中有所展示
    - 终端数据来源于终端管理平台
    - 应用系统数据来源于应用系统管理平台和deops平台
      - deops平台数据主要是组件类软件使用关系台账
      - 应用系统管理平台数据主要是产品类软件使用关系台账
    - 服务器数据来源于服务器管理平台
### 2.3 页面
- 终端使用关系列表页：展示终端使用关系，要到负责人，支持多维度筛选、搜索、排序
- 应用系统使用关系列表页：展示应用系统使用关系，要到负责人，支持多维度筛选、搜索、排序
- 服务器使用关系列表页：展示服务器使用关系，要到负责人，支持多维度筛选、搜索、排序

### 2.4 权限
- 终端使用关系台账：终端负责人本人、超管、终端负责人所属团队负责人
- 应用系统使用关系台账：应用系统负责人、超管、应用系统所属团队负责人
- 服务器使用关系台账：服务器所属应用系统负责人、超管、应用系统所属团队负责人

## 3、可观测需求
### 3.1 软件数据统计大屏

- 软件使用关系统计：
  - 按照终端、服务器、应用系统三个使用场景的使用数据统计（所有人可查看）
  - 每个使用场景下细致的分类统计（团队负责人和超管可查看）
    - 终端
      - 统计维度1：使用网段统计
      - 统计维度2：终端所属团队统计
      - 统计维度3：软件管理类型统计：开源软件、商业软件、自研软件
    - 应用系统
      - 统计维度1：软件类型统计：产品类、组件类（组件类继续下发maven、pypi等）
      - 统计维度2：应用系统所属团队统计
      - 统计维度3：软件管理类型统计：开源软件、商业软件、自研软件
      - 统计维度4：应用系统级别统计
    - 服务器
      - 统计维度1：软件类型统计：按照工具采集的类型统计，pkg、jar、pypi等
      - 统计维度2：软件管理类型统计：开源软件、商业软件、自研软件
      - 统计维度3：服务器所属应用系统所属团队统计
      - 统计维度4：应用系统级别统计
  
- 软件台账统计：（所有人可查看）
  - 统计维度1：产品类、组件类
  - 统计维度2：开源软件、商业软件、自研软件
  - 统计维度3：按照功能类型统计：办公类、开发类、运维管理类、安全类等

    
### 3.2 同步情况
这部分数据仅超管可以查看
- 同步状态统计：统计同步状态，包括同步中、同步成功、同步失败等
- 同步异常情况检测：针对同步过程出现明显报错或者前后两次同步明显数据不一致的情况进行大屏展示、告警、详情下钻展示
- 同步失败详情查看：统计同步失败原因，包括同步失败次数、同步失败原因等
- 数据变动检测：检测数据的新增、修改、删除变更数据的统计分析
